# Advanced Terminal

Десктоп-приложение для macOS, которое собирает множество bash-терминалов в одном окне и привязывает к ним сессии [Claude Code](https://claude.com/claude-code). Именуйте терминалы, раскладывайте их по папкам-вкладкам, следите за статусом каждой сессии Claude одним взглядом и мгновенно возобновляйте работу после перезапуска.

> Electron + React + node-pty + xterm.js. Интерфейс на русском языке.

---

## Зачем это

Когда параллельно работает несколько сессий Claude Code, главная боль — не запустить их, а **понять, какая из них ждёт именно вас**. Advanced Terminal решает это через официальные hooks Claude Code: каждый терминал показывает, что происходит с его сессией — работает, ждёт ввода, просит подтверждения или свободен — а папка-вкладка сводит статусы дочерних терминалов в один приоритетный бейдж.

Идеи и уроки почерпнуты из Crystal, Claude Squad, Warp, opcode, CCManager и официального agent view Claude Code.

## Возможности

### Терминалы и организация
- **Именованные терминалы** — двойной клик по имени переименовывает.
- **Папки-вкладки** — группируйте терминалы по проектам/задачам (по умолчанию: Рабочее, Проекты, Общие вопросы), у папки свой цвет и эмодзи-иконка; создавайте, редактируйте и удаляйте папки, переносите терминалы между ними (меню терминала → «Переместить в папку»).
- **Отдельные окна** — любую папку можно вынести в собственное окно («⧉ Открыть в новом окне») и перетащить на другой монитор; кнопка «Вернуть в главное окно» или закрытие окна возвращает вкладку. Все окна работают с общим состоянием и процессами.
- **Сетка** — 1–3 колонки в зависимости от числа терминалов в папке.
- **Восстановление раскладки** — после перезапуска приложения все терминалы поднимаются заново (тот же id, тот же рабочий каталог).
- **Глобальный поиск (⌘K)** — по именам терминалов, папок, рабочим каталогам и превью прошлых сессий.
- **Поиск по выводу (⌘F)** — по всему scrollback конкретного терминала.

### Интеграция с Claude Code
- **Индикаторы статуса** — `работает` / `ждёт ввода` / `требует разрешения` / `свободен`, обновляются через hooks, а не парсингом вывода.
- **Сводный бейдж папки** — по приоритету: разрешение → ждёт ввода (со счётчиком) → работает.
- **Уведомления macOS** — когда сессия ждёт ввода или завершилась; подавляются, если вы и так смотрите на этот терминал.
- **Привязка сессии к терминалу** — запуск новой (`claude --session-id`), продолжение последней (`claude --continue`) или возобновление привязанной (`claude --resume`). При падении процесса — кнопка «Перезапустить».
- **Браузер прошлых сессий** — читает `~/.claude/projects/`, показывает превью первого сообщения, дату, ветку и число сообщений; открывает сессию в новом терминале в один клик.
- **Дашборд «что требует меня»** — все терминалы всех папок, сгруппированные по статусу, с превью последних строк вывода и полем быстрой отправки команды.

### Git worktrees
- Создание терминала в новом git worktree (своя ветка, свой каталог) с необязательным setup-скриптом (например `npm install`), который выполняется в новом worktree сразу после создания.
- Просмотр diff worktree против основной ветки прямо в приложении.
- Удаление worktree вместе с его терминалом.

### Статистика
- Расход токенов и приблизительная стоимость по каждой сессии (вход / выход / кэш, разбивка по моделям) — считается локально из тех же `~/.claude/projects/*.jsonl`, без обращений к API.

## Как работает определение статуса

Приложение **не парсит вывод терминала** — это хрупко и ломается при обновлениях Claude Code. Вместо этого используется официальный механизм hooks:

1. При старте приложение пишет скрипт `~/.advanced-terminal/hook.sh` и (по кнопке в баннере) регистрирует его в `~/.claude/settings.json` на события `SessionStart`, `UserPromptSubmit`, `Stop`, `Notification`, `PostToolUse`, `SessionEnd`. Существующие чужие hooks не затрагиваются, создаётся резервная копия настроек.
2. Каждый терминал запускается с переменной окружения `ADVTERM_TERM_ID`. Когда Claude Code внутри него дёргает hook, скрипт кладёт файл-событие в спул-каталог `~/.advanced-terminal/events/`.
3. Main-процесс следит за спул-каталогом, разбирает событие, обновляет статус терминала и (при `SessionStart`) автоматически привязывает `session_id`.

Такой подход опирается только на стабильные интерфейсы Claude Code (hooks, `--resume`, локальные логи), поэтому переживает обновления CLI.

## Установка и запуск

Требуется Node.js 20+ и Xcode Command Line Tools (для сборки нативного `node-pty`).

```bash
git clone https://github.com/armeny007/advanced-terminal.git
cd advanced-terminal
npm install
npm run dev          # запуск в режиме разработки
```

Если бинарник Electron не скачался автоматически при `npm install`:

```bash
node node_modules/electron/install.js
```

Прочие команды:

```bash
npm run build        # production-сборка (out/)
npm run dist         # сборка .app / .dmg через electron-builder
npm run typecheck    # проверка типов (main + renderer)
```

При первом запуске нажмите **«Установить»** в жёлтом баннере, чтобы зарегистрировать hooks — без этого не будет индикаторов статуса.

## Архитектура

Три изолированных слоя, единый контракт в `src/shared/types.ts` (типы + IPC-каналы + API, пробрасываемый в renderer через `contextBridge`).

```
src/
  shared/types.ts          Контракт: типы, IPC-каналы, интерфейс window.api
  main/
    index.ts               Точка входа, инициализация модулей
    windows.ts             Главное окно + вынос папок в отдельные окна
    store.ts               Персистентное состояние (userData/state.json), атомарная запись
    pty.ts                 node-pty: жизненный цикл shell, IPC терминалов/папок
    contracts.ts           Интерфейсы Store и PtyManager
    paths.ts, runtime.ts   Пути и общее runtime-состояние
    claude/
      hooks.ts             Генерация hook.sh и установка в ~/.claude/settings.json
      watcher.ts           Слежение за спул-каталогом, статус-машина
      notifications.ts     Уведомления macOS с подавлением по фокусу
      sessions.ts          Парсер ~/.claude/projects для браузера сессий
      usage.ts             Подсчёт токенов/стоимости из JSONL
      worktrees.ts         git worktree через execFile
  preload/index.ts         Безопасный проброс API в renderer
  renderer/src/            React-интерфейс (папки, терминалы, дашборд, модалки)
```

**Модель процессов.** pty-процессы живут вместе с приложением и не переживают его перезапуск (в отличие от tmux) — это осознанный выбор в пользу простоты. Сессии Claude Code при этом не теряются: они возобновляются через `claude --resume <uuid>`, а `session_id` сохраняется в состоянии терминала.
